# Knowledge Graph Document Chat

A production-grade document Q&A system with traceable citations,
document-lifecycle awareness, and a knowledge-graph layer for provenance and
truth-maintenance — built incrementally by a two-person team across **two
repos**: an open-source portfolio starter (Apache 2.0) and a private
commercial product derived from it.

## Documents

- **[goals.md](./goals.md)** — business goals, public/private project
  structure, licensing rationale, business risks, parking lot.
- **[requirements.md](./requirements.md)** — behavioral requirements by
  phase (1–5), with MUST/SHOULD/MAY levels and acceptance criteria.
- **[architecture.md](./architecture.md)** — API-first contract architecture,
  technology choices, repo layout for both repos, sync mechanics, ADR list.
- **[implementation.md](./implementation.md)** — engineering principles
  (test-first, CI), tiers with scope/deliverables/test strategy/DoD, working
  agreements, execution risks, where to start.
